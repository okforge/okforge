# Slides

`openkb deck new` turns wiki content into a polished, **single-file HTML slide deck** ŌĆö
all CSS/JS inlined, no external assets, keyboard-navigable, ready to open
full-screen or share.

> **Real artifact in this folder:** [`attention-intro.html`](attention-intro.html)
> ŌĆö a 9-slide deck generated by `openkb deck new` from one paper, in the
> `openkb-deck-neon` theme. Open it in a browser (ŌåÉ ŌåÆ to navigate, `F` for
> full-screen). It's one self-contained file.

```bash
openkb deck new <name> "<intent>"
```

- `<name>` ŌĆö kebab-case slug; output lands at `output/decks/<name>/index.html`.
- `<intent>` ŌĆö what the deck is about; the generator selects and orders wiki
  content to match.

```bash
openkb deck new attention-intro \
  "A short intro deck on the Transformer and self-attention for an ML reading group"
```

```text
Generating deck 'attention-intro' via skill openkb-deck-neon (default)...
Deck written to output/decks/attention-intro/index.html
```

That command produced [`attention-intro.html`](attention-intro.html). You can also
drive this from inside [`openkb chat`](../chat/) with `/deck new ŌĆ”`.

## Options

| Flag | Effect |
| --- | --- |
| `--skill <name>` | Choose the deck theme/skill (default `openkb-deck-neon`). |
| `--critique` | Run a second-pass critic (`openkb-html-critic`) that reviews and fixes CSS/UX issues before saving. |
| `-y, --yes` | Overwrite an existing deck without prompting (the old one is backed up to `output/decks/<name>-workspace/iteration-N/`). |

```bash
# Warm editorial look, with a quality pass:
openkb deck new okf-pitch "Pitch okforge to a data team" \
  --skill openkb-deck-editorial --critique
```

## Themes

Decks are rendered by a **deck skill** ŌĆö a SKILL.md that defines the visual
direction and a slide grammar. Two ship with okforge (no install needed):

| Skill | Look |
| --- | --- |
| [`openkb-deck-neon`](../../skills/openkb-deck-neon/SKILL.md) *(default)* | Dark "Aurora Glass" ŌĆö near-black background, teal/sky/magenta/amber neon, glassmorphism panels. |
| [`openkb-deck-editorial`](../../skills/openkb-deck-editorial/SKILL.md) | Warm "Editorial Monocle" ŌĆö cream, serif, brick-red accent; a printed-page feel. |

Want a custom look? Drop a third-party deck skill into `~/.config/okforge/skills/` or
`<kb>/skills/` and select it with `--skill` (a same-named skill there overrides
the built-in).

## A grammar keeps decks from degrading

Each deck skill declares a slide grammar that the output must satisfy. The
`openkb-deck-neon` theme requires a `cover` and `closing`, at least 4 distinct
slide types, and no more than 2 of the same type in a row:

```yaml
od:
  mode: deck
  output_path_template: "output/decks/{slug}/index.html"
  deck_grammar:
    kind_attr: data-type
    required: [cover, closing]
    allowed: [cover, chapter, thesis, quote, compare, data, closing]
    min_distinct: 4
    max_consecutive_same: 2
```

[`attention-intro.html`](attention-intro.html) satisfies it with 9 slides spanning
`cover ┬Ę thesis ┬Ę quote ┬Ę compare ┬Ę data ┬Ę closing` ŌĆö which is what stops it from
collapsing into a wall of identical bullet slides.
